Pseudo-code détaillé de run_one_experiment.py
===========================================

Document basé sur le fichier fourni par l'utilisateur.


1) parse_csv_list(raw)
----------------------
Entrée :
- raw : chaîne de caractères contenant une liste séparée par des virgules.
  Exemple : "record_id,age,sex"
  Cette chaîne vient en pratique des arguments CLI, par exemple --public-drop-columns.

Sortie :
- une liste Python de chaînes.
  Exemple : ["record_id", "age", "sex"]

Pseudo-code :
    si raw vaut None :
        retourner []
    sinon :
        découper raw sur les virgules
        enlever les espaces autour de chaque morceau
        ignorer les morceaux vides
        retourner la liste obtenue


2) build_runtime_config(config_path, payload)
---------------------------------------------
Entrées :
- config_path : chemin du fichier JSON de configuration lu sur le disque.
  Exemple : outputs/generated_configs/qi_age-sex__k_5__l_2__t_None__supp_10__arx.json
  Dans ce code, il est passé à la fonction mais n'est pas réellement utilisé pour la logique.

- payload : dictionnaire Python obtenu en lisant le fichier JSON.
  Il contient la configuration "humaine" de départ :
  data, hierarchy_dir, quasi_identifiers, k, l, t, suppression_limit, backend, etc.

Sortie :
- runtime : dictionnaire prêt à être donné à AnonymizationConfig.
  La différence principale est que :
  - data devient un chemin absolu
  - hierarchy_dir est remplacé par hierarchies

Pseudo-code :
    runtime <- copie de payload

    project_root <- dossier parent du dossier scripts

    data_path <- payload["data"]
    si data_path est déjà absolu :
        runtime["data"] <- data_path
    sinon :
        runtime["data"] <- chemin absolu project_root / data_path

    runtime["hierarchies"] <- build_hierarchy_mapping(
        base_dir = project_root,
        hierarchy_dir = payload["hierarchy_dir"],
        quasi_identifiers = payload["quasi_identifiers"]
    )

    supprimer runtime["hierarchy_dir"]

    retourner runtime


3) append_row_to_summary(summary_csv, row)
------------------------------------------
Entrées :
- summary_csv : chemin vers le fichier benchmark_summary.csv.
  C'est le fichier global où on ajoute une ligne par expérience.

- row : dictionnaire représentant une ligne de résumé.
  Il contient par exemple :
  experiment_id, status, config_path, csv_path, eval_csv_path, metrics_path, error, etc.

Sortie :
- aucune valeur retournée.
  La fonction écrit simplement une ligne dans le CSV.

Pseudo-code :
    créer le dossier parent de summary_csv si nécessaire
    vérifier si le fichier existe déjà

    sanitized_row <- sanitize_row_for_csv(row)
        // convertit listes/dicts en texte JSON si besoin

    ouvrir summary_csv en mode append
    créer un DictWriter avec les clés de sanitized_row comme colonnes

    si le fichier n'existait pas :
        écrire l'en-tête

    écrire la ligne sanitized_row


4) _make_initial_row(experiment_id, runtime_config_path, runtime)
-----------------------------------------------------------------
Entrées :
- experiment_id : identifiant unique de l'expérience.
  Exemple : qi_age-sex__k_5__l_2__t_None__supp_10__arx

- runtime_config_path : chemin du fichier JSON runtime sauvegardé dans outputs/configs/.
  C'est la config exacte réellement exécutée.

- runtime : dictionnaire runtime complet de l'expérience.
  Il contient notamment quasi_identifiers, k, l, t, suppression_limit, backend.

Sortie :
- un dictionnaire row initial, qui servira de base pour le benchmark_summary.csv.
  Au départ, le statut est "started" et les chemins de sortie sont vides.

Pseudo-code :
    retourner {
        experiment_id,
        status = "started",
        timestamp = timestamp(),
        config_path = runtime_config_path,
        quasi_identifiers = concaténation des QI avec "|",
        k, l, t,
        suppression_limit,
        backend,
        csv_path = "",
        eval_csv_path = "",
        metrics_path = "",
        error = ""
    }


5) _run_anonymization(runtime)
------------------------------
Entrée :
- runtime : dictionnaire runtime prêt à être utilisé par RECITALS.
  Il contient tout ce qu'il faut pour construire AnonymizationConfig.

Sortie :
- result : objet retourné par AnonymizationManager.anonymize(...)
  Cet objet permet ensuite de récupérer :
  - le dataframe anonymisé
  - les métriques d'anonymisation

Pseudo-code :
    config <- AnonymizationConfig(**runtime)
    result <- AnonymizationManager.anonymize(config)
    retourner result


6) run_one_experiment(...)
--------------------------
Entrées :
- runtime : dictionnaire runtime complet, déjà prêt pour l'exécution.
  C'est la vraie config exécutée par le manager.

- experiment_id : nom unique de l'expérience.
  Sert à nommer les fichiers produits : CSV, JSON, erreurs.

- output_root : dossier racine où tout sera écrit.
  Exemple : outputs/

- save_anonymized_csv : booléen.
  True = on sauvegarde la version publique dans outputs/anonymized/
  False = on ne la sauvegarde pas.

- save_anonymized_eval_csv : booléen.
  True = on sauvegarde la version interne d'évaluation dans outputs/anonymized_eval/
  False = on ne la sauvegarde pas.

- public_drop_columns : liste de colonnes à supprimer seulement du CSV public.
  Exemple typique : ["record_id"]
  Ainsi, le public ne voit pas record_id, mais la version eval peut encore l'avoir.

- append_summary : booléen.
  True = on ajoute la ligne finale dans benchmark_summary.csv
  False = on n'écrit pas cette ligne

Sortie :
- details : dictionnaire riche contenant :
  - experiment_id
  - row
  - runtime
  - runtime_config_path
  - metrics_path
  - public_csv_path
  - eval_csv_path
  - anonymized_df
  - public_df
  - error_path

Pseudo-code :
    public_drop_columns <- [] si None

    créer / garantir les dossiers :
        output_root/
        output_root/metrics/
        output_root/configs/
        output_root/anonymized/
        output_root/anonymized_eval/
        output_root/errors/

    summary_csv <- output_root/benchmark_summary.csv

    runtime_config_path <- output_root/configs/experiment_id.json
    sauvegarder runtime dans runtime_config_path

    row <- _make_initial_row(experiment_id, runtime_config_path, runtime)

    details <- dictionnaire interne qui stockera tous les résultats utiles

    essayer :
        result <- _run_anonymization(runtime)

        anonymized_df <- None
        public_df <- None
        public_csv_path <- output_root/anonymized/experiment_id.csv
        eval_csv_path <- output_root/anonymized_eval/experiment_id.csv

        si save_anonymized_csv OU save_anonymized_eval_csv :
            anonymized_df <- result.get_anonymized_data_as_dataframe()

            si save_anonymized_eval_csv :
                sauvegarder anonymized_df dans eval_csv_path
                mettre à jour row["eval_csv_path"]
                mettre à jour details["eval_csv_path"]

            si save_anonymized_csv :
                public_df <- anonymized_df après suppression des colonnes public_drop_columns
                sauvegarder public_df dans public_csv_path
                mettre à jour row["csv_path"]
                mettre à jour details["public_csv_path"]

        metrics <- collect_result_metrics(result)
        metrics <- to_jsonable(metrics)

        ajouter dans metrics :
            experiment_id
            status = "success"
            public_drop_columns
            public_csv_path
            eval_csv_path

        metrics_path <- output_root/metrics/experiment_id.json
        sauvegarder metrics dans metrics_path

        row["status"] <- "success"
        row["metrics_path"] <- metrics_path
        fusionner row avec les metrics

        mettre à jour details avec :
            metrics
            metrics_path
            anonymized_df
            public_df

        afficher les chemins créés

    sauf exception exc :
        error_txt <- traceback complet
        error_path <- output_root/errors/experiment_id.txt
        sauvegarder error_txt

        row["status"] <- "failed"
        row["error"] <- message de l'exception
        details["error_path"] <- error_path

        afficher [FAILED] et le traceback

    si append_summary :
        append_row_to_summary(summary_csv, row)

    retourner details


7) run_one_experiment_from_config(...)
--------------------------------------
Entrées :
- config_path : chemin vers le fichier JSON d'expérience.
  C'est la config utilisateur / benchmark qui sera d'abord lue puis transformée en runtime.

- output_root : dossier racine où écrire tous les résultats.
  Exemple : outputs/

- name : nom optionnel pour forcer l'identifiant de l'expérience.
  Si None, l'identifiant est construit automatiquement avec make_experiment_id(...)

- save_anonymized_csv : booléen.
  Indique si on sauvegarde la version publique du CSV anonymisé.

- save_anonymized_eval_csv : booléen.
  Indique si on sauvegarde la version interne d'évaluation du CSV anonymisé.

- public_drop_columns : liste optionnelle des colonnes retirées du CSV public seulement.
  Exemple : ["record_id"]

- append_summary : booléen.
  Indique si on ajoute la ligne finale dans benchmark_summary.csv.

Sortie :
- le dictionnaire details renvoyé par run_one_experiment(...)

Pseudo-code :
    config_path <- chemin absolu de config_path

    payload <- load_json(config_path)
    runtime <- build_runtime_config(config_path, payload)

    si name est fourni :
        experiment_id <- name
    sinon :
        experiment_id <- make_experiment_id(
            quasi_identifiers = runtime["quasi_identifiers"],
            k = runtime["k"],
            l = runtime["l"],
            t = runtime["t"],
            suppression_limit = runtime["suppression_limit"],
            backend = runtime["backend"]
        )

    retourner run_one_experiment(
        runtime = runtime,
        experiment_id = experiment_id,
        output_root = output_root,
        save_anonymized_csv = save_anonymized_csv,
        save_anonymized_eval_csv = save_anonymized_eval_csv,
        public_drop_columns = public_drop_columns,
        append_summary = append_summary
    )


8) main()
---------
Entrée :
- pas d'argument Python direct
- mais la fonction lit les arguments de la ligne de commande

Arguments CLI :
- --config : chemin du fichier JSON d'expérience à lancer
- --output-root : dossier racine de sortie
- --name : nom optionnel de l'expérience
- --save-anonymized-csv : drapeau pour sauvegarder le CSV public
- --save-anonymized-eval-csv : drapeau pour sauvegarder le CSV eval
- --public-drop-columns : chaîne de colonnes séparées par des virgules

Sortie :
- aucune valeur retournée directement
- la fonction lance juste run_one_experiment_from_config(...)

Pseudo-code :
    parser <- créer ArgumentParser
    définir tous les arguments CLI
    args <- parser.parse_args()

    run_one_experiment_from_config(
        config_path = args.config,
        output_root = args.output_root,
        name = args.name,
        save_anonymized_csv = args.save_anonymized_csv,
        save_anonymized_eval_csv = args.save_anonymized_eval_csv,
        public_drop_columns = parse_csv_list(args.public_drop_columns),
        append_summary = True
    )


9) Résumé global du flux
------------------------
Pseudo-code très court :

    lire une config JSON
    -> construire une config runtime
    -> générer experiment_id
    -> lancer l'anonymisation
    -> récupérer le dataframe anonymisé
    -> sauvegarder éventuellement :
         - version publique (anonymized)
         - version d'évaluation (anonymized_eval)
    -> extraire les métriques
    -> sauvegarder metrics.json
    -> ajouter une ligne dans benchmark_summary.csv
    -> retourner un dictionnaire details


10) Différence entre les deux CSV produits
------------------------------------------
- anonymized :
  version publique
  = on peut supprimer certaines colonnes avec public_drop_columns
  exemple : retirer record_id

- anonymized_eval :
  version interne d'évaluation
  = dataframe anonymisé complet sauvegardé tel quel
  = utilisé ensuite par les attaques pour vérifier la vérité terrain
